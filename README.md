# [Wio](https://wio.onl/)

Wio is a backend-as-a-service platform for developing web applications with AI agents.

- **Easy-to-learn**: The entire development cycle consists of a few CLI commands.
- **Instant deployment**: Sites are deployed instantly with zero configuration.
- **AI-agent-powered**: All development is powered by an AI agent.

Wio provides a [managed database](https://wio.onl/docs/features/database), [websockets](https://wio.onl/docs/features/websockets), [observability](https://wio.onl/docs/features/observability), [markdown renderer](https://wio.onl/docs/features/markdown-renderer), [sound player](https://wio.onl/docs/features/sound-player), and [LLM API](https://wio.onl/docs/features/llm-api). Sites using Wio rely on the [SDK](https://wio.onl/docs/sdk) to interface the backend services provided by Wio. The development of sites is meant to be fully AI driven, requiring no human-written code.

## Getting Started

1. Install the [Wio CLI](https://www.npmjs.com/package/wio-cli) NPM package:

```bash
npm i -g wio-cli
```

2. Register a new user tag by visiting [https://wio.onl/register](https://wio.onl/register).

> [!NOTE]  
> Be sure to save your user tag someplace safe! You will need it to login and manage your sites.

3. Login using your user tag:

```bash
wio login <user-tag>
```

> [!NOTE]  
> After logging in, you can see your user tag by using `wio status`.

4. Initialize a new site (the site name must be unique):

```bash
wio init my-site
```

5. Switch to the site directory:

```bash
cd my-site
```

6. Launch an AI agent of your choice (Claude Code, Codex, and OpenCode are supported) and prompt it to create a web application of your choice. The agent will automatically infer the instructions from the pre-packaged [SDK](https://wio.onl/docs/sdk) to use Wio infrastructure.

> [!NOTE]  
> The user is responsible for bringing their own AI coding agent.

7. Once the agent has finished coding, deploy the website:

```bash
wio push
```

8. The site is immediately live at https://my-site.wio.onl.

## Core Workflows

To fully evaluate Wio's capabilities, TAs can test the following workflow which highlights the CLI, SDK, Database, and Observability features.

1. **Setup & Initialization**
   - After installing the Wio CLI locally (`npm i -g wio-cli`) and logging in, initialize a new site.
   - Run `wio init my-test-site` and navigate into the `my-test-site` directory.

2. **Developing with the SDK**
   - The CLI will seed a default `index.html` file into the folder.
   - Prompt an AI Agent to create a web application of your choice.

   _Example prompts to try with your AI Agent:_

   > "Use Wio to build a real-time multiplayer drawing game where players take turns drawing an AI-generated word on a shared canvas, keeping track of everyone's scores."

   > "Use Wio to create an AI-powered blog. When I provide a short topic, automatically generate and publish a fully formatted markdown article, saving all posts for visitors to read."

3. **Instant Deployment**
   - Make sure you have registered (`https://wio.onl/register`) and logged in (`wio login <user-tag>`).
   - Run `wio push`.
   - Your site is immediately deployed to `https://my-test-site.wio.onl`.

4. **Observability Tracking**
   - Visit the deployed URL (`https://my-test-site.wio.onl`) and generate some traffic by interacting with the page.
   - Go to your Wio Dashboard (`https://wio.onl/dashboard`) to see real-time observability metrics. The platform captures HTTP requests, WebSocket interactions, and any AI LLM telemetry generated by your application's SDK calls.

## Deliverable 3 Improvements & D2 Feedback

In D2, we received feedback that our documentation links were broken and our product vision (e.g., the Gemma AI demo) felt detached from the core offering.

**Key Changes:**

1. **Docusaurus Unification**: All disjointed docs, architecture notes, and getting started guides have been merged into a single Docusaurus site strictly hosted at `/docs/`. This ensures developers have a cohesive manual alongside our stable domain mapping.

3. **SDK & CLI Maturation**: We added CLI authentication (`wio login`) and expanded the frontend SDK (audio player, modals, DB helpers) so developers (and AI agents) have robust tools to scaffold and connect UI to our backend APIs.

## Architecture

Wio is written in TypeScript and uses [Bun](https://bun.com/) as the runtime. The backend is powered by [Fastify](https://fastify.dev/). Data persistence is implemented with [PostgreSQL](https://www.postgresql.org/) and static file storage is done with [MinIO](https://github.com/minio/minio). The project relies heavily on separation of concerns using the [Repository Pattern](https://www.cosmicpython.com/book/chapter_02_repository.html) and the [Controller pattern](https://en.wikipedia.org/wiki/Model%E2%80%93view%E2%80%93controller).

Wio consists of three parts:

1. **Backend**: provides services to the client sites.
2. **CLI**: command line interface for managing sites.
3. **SDK**: interface for accessing Wio services.

See the [architecture guide](ARCHITECTURE.md) for more information.
